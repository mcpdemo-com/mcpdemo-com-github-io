What Is This?

Claude is a powerful AI — but without a live connection to the web, it works from a snapshot of the world that stopped updating the moment its training ended. Web Search Model Context Protocol (MCP) integration changes that entirely.

This demo connects Claude to websearch_web_search, a live search tool hosted on the mcpcio.com MCP infrastructure at https://api.mcpcio.com/mcp. Every query triggers an actual web search — not a cached result, not a training-time approximation. Real data, returned at the moment you ask.

Why Static Training Data Isn't Enough

Every large language model, including Claude, is trained on a dataset with a hard cutoff date. That cutoff means Claude genuinely doesn't know what happened last week, what a product costs today, or which software library released a breaking update yesterday. MCP solves this by giving Claude a live retrieval path — Claude queries the web in real time and returns answers grounded in what's actually true right now.

How It Works

Three steps, one live result

01

You send a query

Type any question into the demo. The demo page sends it to a Cloudflare Worker at api.mcpdemo.com/proxy/web-search.

02

Worker calls the MCP tool

The Worker calls websearch_web_search directly on the mcpcio.com MCP server — no Claude in the middle, no LLM interpretation of the search itself.

03

Live results come back

The MCP server runs a real web search and returns a synthesized answer with source attribution. You see what the web actually says right now.

Direct MCP Call Pattern

// Direct MCP tool call — no LLM middleman
const resp = await fetch('https://api.mcpcio.com/mcp', {
  method: 'POST',
  headers: {
    'Content-Type':  'application/json',
    'Authorization': 'Bearer YOUR_MCPCIO_API_KEY',
  },
  body: JSON.stringify({
    jsonrpc: '2.0', id: 1,
    method:  'tools/call',
    params: {
      name:      'websearch_web_search',
      arguments: { query: 'your search query', num_results: 5 }
    }
  })
});
